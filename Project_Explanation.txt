================================================================================
Vansec: Intrusion Detection System (IDS) for VANET
================================================================================

--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
Project Goal: Develop an IDS to detect and mitigate attacks (e.g., blackhole, DDoS, and Sybil) in Vehicular Ad-Hoc Networks (VANETs).

VANETs enable vehicles to communicate with each other (V2V) and with infrastructure (V2X) to improve road safety and traffic efficiency. However, these networks are highly vulnerable to cyberattacks. The Vansec project aims to build a robust Machine Learning-based Intrusion Detection System that analyzes network traffic and vehicle dynamics to accurately classify normal traffic and identify malicious activities (attacks). 

--------------------------------------------------------------------------------
2. THE DATA LEAKAGE PROBLEM AND REMOVING LEAKY COLUMNS
--------------------------------------------------------------------------------
In machine learning, "Data Leakage" occurs when a model is trained using information that it would not realistically have access to in a real-world, deployed environment. This causes the model to "cheat" and achieve artificially high performance (like scoring 100% accuracy) during testing, but fail completely in the real world.

In our initial dataset, we identified and removed several leaky columns:
- Pre-computed Labels: Columns like 'sybil_score' or 'blackhole_score' were directly derived from the target label itself.
- Attacker-side Counters: Columns like 'my_attack_packets' are only known to the attacker. A defending IDS monitoring the network would never see this variable.
- Simulation Artifacts: Variables like 'timestamp' were acting as a monotonic simulation clock. Because attacks were injected at specific times in the simulation, the model learned to predict attacks based on the clock time rather than actual network behavior.

By removing these leaky columns, we forced the model to learn genuine, observable patterns (like vehicle speed and packet size) rather than relying on unrealistic "cheat codes" provided by the simulation environment.

--------------------------------------------------------------------------------
3. MUTUAL INFORMATION FOR FEATURE SELECTION
--------------------------------------------------------------------------------
Mutual Information (MI) is a statistical metric that measures how much knowing the value of one variable (a feature) reduces uncertainty about another variable (the target label). Unlike simple correlation, which only finds linear relationships, MI can capture complex, non-linear dependencies.

How it was used:
We applied Mutual Information scoring to rank all our remaining features based on how much information they provided about the attack type. Features with an MI score near zero were deemed useless and discarded. We kept the top features (such as 'avg_speed', 'distance_moved', and 'speed_mps') that showed a strong relationship with the target. This streamlined our model, making it faster and less prone to overfitting on noisy data.

--------------------------------------------------------------------------------
4. TRAIN/TEST SPLITTING: SOLVING THE 100% ACCURACY ISSUE
--------------------------------------------------------------------------------
Initially, our model achieved a suspicious 100% accuracy. A major culprit was how we split our data into training and testing sets. 

The Problem (Row-based splitting): 
We initially split the data randomly row-by-row. However, each unique vehicle ('car_id') generated thousands of rows of data over time. A random split scattered rows from the exact same car into both the training and testing sets. The model simply memorized the specific behavioral fingerprints of individual cars it saw in training, rather than learning general, overarching attack patterns. This is known as "Vehicle Identity Leakage."

The Solution (Stratified Group Splitting):
We realized we needed to ensure no 'car_id' overlap between the training and testing sets. We implemented a split that groups all rows by their 'car_id' first. If Car #5 goes into the training set, every single row generated by Car #5 goes into training. If Car #12 goes into testing, all its rows go into testing. Furthermore, we stratified this split to ensure that every type of attack (especially minority classes like DDoS) had cars represented in both sets. After implementing this, the model behaved correctly and yielded a realistic, trustworthy accuracy.

--------------------------------------------------------------------------------
5. RANDOM FOREST AND XGBOOST: MODEL SELECTION
--------------------------------------------------------------------------------
We chose two powerful tree-based ensemble algorithms for this task:
- Random Forest: Builds hundreds of independent decision trees using random subsets of data and features, and averages their predictions. 
- XGBoost (Extreme Gradient Boosting): Builds trees sequentially, where each new tree tries to correct the errors made by the previous trees.

Why we chose them:
Tree-based models excel at tabular data. They require very little feature scaling, handle non-linear relationships naturally, and provide excellent feature importance interpretability. 

Why Random Forest behaved better:
In our final tests, Random Forest slightly outperformed XGBoost (82.6% vs 78.9% accuracy). Random Forest's method of building independent trees makes it highly robust to noise and less prone to overfitting. XGBoost's sequential error-correction can sometimes cause it to overfit to residual noise in synthetic datasets, which likely gave Random Forest the edge in generalizing the underlying traffic patterns.

--------------------------------------------------------------------------------
6. RESULTS AND INFERENCE DEMO
--------------------------------------------------------------------------------
Metrics Explained:
- Precision: Out of all the instances the model predicted as an attack, how many were ACTUALLY attacks? (Measures false positives).
- Recall: Out of all the actual attacks that occurred, how many did the model successfully find? (Measures false negatives).
- F1-Score: The harmonic mean of Precision and Recall. It provides a single metric that balances both, which is especially useful for imbalanced datasets.

Final Results (Random Forest):
- Overall Accuracy: 82.64%
- F1 Weighted: 82.70%
- F1 Macro: 71.01%
